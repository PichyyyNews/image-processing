# โครงการสอน (Course Outline) - สัปดาห์ที่ 5
**หัวข้อ:** การตรวจจับโครงร่างและการสกัดภาพแยกชิ้นอัตโนมัติ (Contour Detection & Smart ROI Auto-Cropper)
**เวลาเรียน:** 5 ชั่วโมง (บรรยาย 2 ชั่วโมง / ปฏิบัติการ 3 ชั่วโมง)
**เครื่องมือที่ใช้:** VS Code, Miniconda, OpenCV, Python

---

## 1. วัตถุประสงค์การเรียนรู้ (Learning Objectives)
เมื่อสิ้นสุดการเรียนในสัปดาห์นี้ นักศึกษาจะสามารถ:
1. **เข้าใจความแตกต่าง** ระหว่างเส้นขอบ (Edge) และโครงร่าง (Contour) ในการประมวลผลภาพดิจิทัล
2. **ใช้งานฟังก์ชัน OpenCV พื้นฐาน** ที่เกี่ยวข้องกับ Contour ได้อย่างคล่องแคล่ว (เช่น `cv2.findContours()`, `cv2.drawContours()`)
3. **วิเคราะห์คุณสมบัติของวัตถุ** จาก Contour ได้ เช่น การหาจุดศูนย์ถ่วง (Moments), ขนาดพื้นที่ (Area), และความยาวรอบรูป (Perimeter)
4. **ตีกรอบล้อมรอบวัตถุ (Bounding Box)** เพื่อหาพิกัดและขนาดของวัตถุที่สนใจ
5. **พัฒนาโปรแกรมอัตโนมัติ (Automation Script)** เพื่อตัดภาพวัตถุ (Crop) จากภาพขนาดใหญ่แยกเป็นไฟล์ย่อยๆ สำหรับนำไปทำ Dataset ในสัปดาห์ต่อไป

## 2. แผนการสอนช่วงบรรยาย (Lecture Plan) - 2 ชั่วโมง

### 2.1 ทบทวนความรู้เดิม (15 นาที)
* ทบทวนเรื่อง Edge Detection จากสัปดาห์ที่ 4 (Canny, Sobel)
* อภิปรายข้อจำกัดของ Edge (เส้นขอบอาจขาดหาย ไม่ต่อเนื่อง) นำไปสู่ความจำเป็นในการหา Contour (ขอบเขตปิดที่ชัดเจน)

### 2.2 ทฤษฎี Contour Detection (45 นาที)
* **Contour คืออะไร?** อธิบายหลักการทางคณิตศาสตร์เบื้องหลังการหาพิกัดต่อเนื่องที่มีค่าความเข้มสีเท่ากัน
* **Image Moments:** การสกัดคุณลักษณะเชิงพื้นที่ (Spatial characteristics) เพื่อหาตำแหน่งศูนย์กลาง (Centroid) และขนาด (Area) ของวัตถุ
* **Hierarchy:** โครงสร้างความสัมพันธ์ของ Contour (พ่อ-ลูก / วัตถุซ้อนวัตถุ)

### 2.3 การประยุกต์ใช้เพื่อตีกรอบวัตถุ (Bounding Box) (45 นาที)
* ความแตกต่างระหว่าง `cv2.boundingRect()` (กรอบสี่เหลี่ยมตั้งตรง) และ `cv2.minAreaRect()` (กรอบสี่เหลี่ยมที่หมุนตามองศาวัตถุ)
* ทำไมถึงต้องตีกรอบวัตถุ? (เพื่อตีกรอบ ROI - Region of Interest เตรียมส่งให้ AI Model)

### 2.4 ถาม-ตอบและสรุป (15 นาที)
* เปิดโอกาสให้นักศึกษาซักถามข้อสงสัยในภาคทฤษฎีก่อนเริ่มลงมือปฏิบัติ

## 3. แผนการสอนช่วงปฏิบัติการ (Lab Plan) - 3 ชั่วโมง
**Lab 5: Smart ROI Dataset Auto-Cropper**

### 3.1 การเตรียมสภาพแวดล้อมและข้อมูล (30 นาที)
* เปิด VS Code และ Activate Environment `dip_env` ที่สร้างไว้จากสัปดาห์ที่ 1
* แจกจ่ายภาพตัวอย่าง (เช่น ภาพสายพานลำเลียงสินค้าที่มีวัตถุหลายชิ้นในภาพเดียว) ให้กับนักศึกษา

### 3.2 หา Contour และวาดเส้นโครงร่าง (45 นาที)
* ให้นักศึกษาเขียนโค้ด Preprocessing: แปลงภาพเป็น Grayscale -> ทำ Thresholding หรือ Canny Edge
* ใช้ฟังก์ชัน `cv2.findContours()` เพื่อหาพิกัดเส้นขอบเขตทั้งหมด
* ใช้ฟังก์ชัน `cv2.drawContours()` เพื่อวาดเส้นสีทับลงบนภาพต้นฉบับเพื่อตรวจสอบความถูกต้อง

### 3.3 กรองวัตถุขยะด้วยขนาดพื้นที่ (Area Filtering) (45 นาที)
* ใช้ฟังก์ชัน `cv2.contourArea()` เพื่อคำนวณขนาดของแต่ละ Contour
* กำหนดเงื่อนไข `if area > threshold:` เพื่อตัดชิ้นส่วนที่เป็นเพียง Noise หรือวัตถุขนาดเล็กเกินไปทิ้งไป

### 3.4 ตัดรูปและบันทึกอัตโนมัติ (Auto-Cropper) (60 นาที)
* วนลูป (For loop) เข้าถึงแต่ละ Contour ที่ผ่านการกรองขนาดแล้ว
* ใช้ `cv2.boundingRect()` เพื่อดึงค่าพิกัด `(x, y, w, h)`
* ใช้ Array Slicing ของ NumPy `cropped_img = image[y:y+h, x:x+w]` เพื่อทำการ Crop
* ใช้ `cv2.imwrite()` เซฟเป็นไฟล์ใหม่ (เช่น `object_1.jpg`, `object_2.jpg`) เข้าสู่โฟลเดอร์แยกโดยอัตโนมัติ
* **Challenge (การบ้านเสริม):** ถ่ายภาพวัตถุบนโต๊ะตัวเอง และเขียนสคริปต์ให้ทำงานสมบูรณ์กับรูปภาพของตนเอง
