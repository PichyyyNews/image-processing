## 1. คำอธิบายรายวิชา (Course Description)
ศึกษาและปฏิบัติเกี่ยวกับหลักการ ทฤษฎีภาพดิจิทัล การประมวลผลภาพเชิงดิจิทัลเบื้องต้น การปรับปรุงคุณภาพภาพในโดเมนเชิงพื้นที่ การปรับปรุงภาพในโดเมนเชิงความถี่ การซ่อมคืนสภาพภาพ การประมวลผลภาพสีในระดับพิกเซล การบีบอัดภาพ การแบ่งส่วนภาพ การหาขอบหาจุดเด่นภายในภาพ และการจับความเคลื่อนไหว

## 2. ผลลัพธ์การเรียนรู้ระดับรายวิชา (Course Learning Outcomes - CLO)
1. **CLO 1**: อธิบายสถาปัตยกรรมและหลักการทำงานของคอมพิวเตอร์วิทัศน์รุ่นดั้งเดิมเทียบกับแบบจำลอง Deep Learning ได้
2. **CLO 2**: พัฒนาทักษะการเขียนโปรแกรม Python แบบ Scripting (.py) โดยใช้ VS Code ในการจัดการระบบไฟล์ จัดการสิ่งแวดล้อม (Venv) และเตรียมข้อมูลภาพได้อย่างเป็นระบบ
3. **CLO 3**: สามารถฝึกฝน ปรับแต่ง และนำโมเดล Computer Vision (CNN, YOLO, MediaPipe) ไปประยุกต์ใช้งานจริงบนสภาพแวดล้อมจำลองส่วนบุคคลผ่าน VS Code ได้
4. **CLO 4**: ปฏิบัติงานด้วยความรอบคอบ ปลอดไข ตระหนักถึงจริยธรรมในการเขียนโค้ด การจัดเก็บข้อมูลภาพ และการรักษาความเป็นส่วนตัวของข้อมูลผู้ใช้งาน

## 3. กลยุทธ์การขจัดบัคเรื่องสภาพแวดล้อมในแต่ละเครื่องของนักศึกษา (Crucial Setup Strategy)
เพื่อป้องกันปัญหาเครื่องของนักศึกษามีระบบปฏิบัติการ (Windows, macOS Intel, macOS Apple Silicon, Linux) หรือเวอร์ชัน Python ที่แตกต่างกันจนเกิดข้อผิดพลาดขณะติดตั้งไลบรารีขนาดใหญ่ เช่น OpenCV และ PyTorch เราจะใช้แนวทางดังนี้:
1.  **ไม่ใช้ Python ทั่วไปร่วมกับ pip**: ป้องกันการชนกันของไลบรารีระดับระบบปฏิบัติการ และเลี่ยงข้อผิดพลาด "Microsoft Visual C++ 14.0 is required" บนเครื่อง Windows
2.  **ใช้ Miniconda (Conda Engine)**: Miniconda เป็นตัวจัดการแพ็คเกจที่ดาวน์โหลดไฟล์ binary สำเร็จรูปที่คอมไพล์มาแล้วตามแต่ละ OS ทำให้ติดตั้ง PyTorch และ OpenCV สำเร็จได้ง่ายโดยไม่ต้องใช้ Build Tools ท้องถิ่น
3.  **การใช้ไฟล์ `environment.yml` ล็อกเวอร์ชัน**: ครูผู้สอนจะแจกไฟล์คอนฟิกูเรชันหนึ่งไฟล์ เพื่อให้นักศึกษาใช้สร้างสภาพแวดล้อมเหมือนกันทุกเครื่อง 100% ภายในคำสั่งเดียว

## 4. แผนการจัดหน่วยเรียนรู้รายสัปดาห์ (15-Week Schedule with Miniconda Setup)
จัดการเรียนการสอน 5 ชั่วโมงต่อสัปดาห์ (บรรยาย 2 ชม. + ปฏิบัติ 3 ชม.)

### สัปดาห์ที่ 1: บทนำสู่ภาพดิจิทัลและการติดตั้งสภาพแวดล้อมด้วย Miniconda และ VS Code
*   **บรรยาย (2 ชั่วโมง)**:
    *   ความสัมพันธ์ระหว่าง Image Processing, Computer Vision และโมเดล AI ยุคใหม่
    *   ทฤษฎีภาพดิจิทัล: Pixels, Channels (BGR), Resolution, Bit depth 
    *   ความสำคัญของการทำ Virtual Environment ในงานวิศวกรรมคอมพิวเตอร์วิทัศน์
*   **ปฏิบัติการ (3 ชั่วโมง) - LAB 1: Standardized Environment Setup**:
    *   ติดตั้ง **VS Code** และส่วนขยาย Python / Pylance
    *   ติดตั้ง **Miniconda** ตามระบบปฏิบัติการของตนเอง
    *   สร้างไฟล์คอนฟิกูเรชันร่วมกันชื่อ `environment.yml` (ดูสเปกไฟล์ในหัวข้อที่ 6)
    *   เปิด Terminal ใน VS Code และสร้างสภาพแวดล้อมเสมือนอัตโนมัติด้วยคำสั่ง:
        ```bash
        conda env create -f environment.yml
        ```
    *   เปิดใช้งานสภาพแวดล้อมเสมือน (`conda activate dip_env`) และเลือก Python Interpreter ของ VS Code ไปที่ `dip_env`
    *   เขียนโค้ดเปิดภาพ แสดงผล และเซฟภาพทดสอบการเชื่อมต่อ OpenCV
*   **เครื่องมือหลัก**: VS Code, Miniconda
*   **ฟังก์ชัน OpenCV**: `cv2.imread()`, `cv2.imshow()`, `cv2.imwrite()`, `cv2.cvtColor()`
*   **งานปฏิบัติ**: สร้าง Workspace ใน VS Code รันสคริปต์ตรวจสอบเวอร์ชัน Python, OpenCV, PyTorch ผ่าน Terminal เพื่อยืนยันว่าตรงกันทุกเครื่อง

### สัปดาห์ที่ 2: การประมวลผลพิกเซลและการดำเนินการเรขาคณิต (Image Manipulation in VS Code)
*   **บรรยาย (2 ชั่วโมง)**:
    *   ความสำคัญของการทำ Image Preprocessing และการขยายปริมาณข้อมูล (Data Augmentation)
    *   คณิตศาสตร์ระดับพิกเซลและการเข้าถึงข้อมูลภาพแบบ Array Slice โดยใช้ NumPy
    *   การแปลงสัดส่วนภาพและทรานส์ฟอร์ม: Resizing, Cropping, Flipping, และการหมุนภาพ (Rotation)
*   **ปฏิบัติการ (3 ชั่วโมง) - LAB 2: Batch Image Processing Script**:
    *   ฝึกการดีบักโค้ด (Debugging) ใน VS Code โดยการปักจุด Breakpoint เพื่อดูค่าพิกเซลและโครงสร้าง Matrix ภาพ
    *   การเขียนสคริปต์ Python ใน VS Code เพื่อรับรูปภาพต้นแบบ จากนั้นประมวลผลย่อภาพ ครอบภาพเฉพาะส่วน (ROI) และสุ่มหมุน/กลับภาพเซฟเป็นไฟล์ใหม่
*   **เครื่องมือหลัก**: VS Code, Miniconda
*   **ฟังก์ชัน OpenCV**: `cv2.resize()`, `cv2.flip()`, `cv2.getRotationMatrix2D()`, `cv2.warpAffine()`
*   **งานปฏิบัติ**: เขียนโปรแกรมสคริปต์วนลูปประมวลผลภาพในไดเรกทอรีอัตโนมัติ (Batch script) นำภาพสินค้ามาเตรียมขนาด (Resize) เป็น 224x224 พิกเซล และกลับด้านรูปภาพทั้งหมดเพื่อเป็นข้อมูลตัวอย่างป้อนโมเดล AI

### สัปดาห์ที่ 3: การจัดการแสง สี และการทำความสะอาดข้อมูลภาพ (Contrast & Filtering)
*   **บรรยาย (2 ชั่วโมง)**:
    *   การปรับปรุงแสงและความคมชัด (Brightness & Contrast) เพื่อลดผลกระทบจากแสงแวดล้อมที่ไม่แน่นอน
    *   กราฟฮิสโตแกรมและการปรับความต่างแสงเฉลี่ยด้วย CLAHE
    *   ปัญหาและรูปแบบของสัญญาณรบกวน (Noise): Gaussian Noise และ Salt & Pepper Noise
    *   ทฤษฎีการกรองภาพเชิงพื้นที่ (Spatial Filtering): Gaussian Blur, Median Blur, และ Bilateral Filter
*   **ปฏิบัติการ (3 ชั่วโมง) - LAB 3: Enhancement & Filtering Pipeline**:
    *   การเขียนโค้ดวิเคราะห์ฮิสโตแกรมและพล็อตกราฟแยกแชนเนลสีใน VS Code
    *   การสร้างตัวกรองกำจัด Noise เพื่อลบจุดเม็ดทรายที่ปนเปื้อนในภาพ
    *   การใช้ CLAHE ปรับสมดุลความสว่างของภาพถ่ายมุมมืด
*   **เครื่องมือหลัก**: VS Code, Miniconda
*   **ฟังก์ชัน OpenCV**: `cv2.createCLAHE()`, `cv2.GaussianBlur()`, `cv2.medianBlur()`, `cv2.bilateralFilter()`
*   **งานปฏิบัติ**: เขียนสคริปต์ประมวลผลล้าง Noise ภาพถ่ายวัตถุจากแสงน้อยให้สะอาดชัดเจน (Preprocessing Pipeline) เพื่อพร้อมรองรับการอ่านรหัส QR code หรือใบหน้า

### สัปดาห์ที่ 4: การสกัดเส้นขอบและการดัดแปลงสัณฐานวิทยา (Edge & Morphology)
*   **บรรยาย (2 ชั่วโมง)**:
    *   คณิตศาสตร์การหาอนุพันธ์ของภาพเพื่อระบุความแตกต่าง (Edges): Sobel, Laplacian, และ Canny Edge Detection
    *   ทฤษฎีสัณฐานวิทยา (Morphological Operations): Erosion (กร่อนภาพ), Dilation (ขยายภาพ)
    *   การประยุกต์ใช้ Opening/Closing เพื่อเชื่อมรอยแยกหรือลบรอยเปื้อนจุดเล็ก ๆ
*   **ปฏิบัติการ (3 ชั่วโมง) - LAB 4: Custom Object Edge Extraction**:
    *   เขียนโค้ดสร้างสคริปต์ตรวจจับขอบบนภาพถ่ายด้วย Canny Edge Detection
    *   การปรับปรุงเส้นขอบวัตถุให้หนาขึ้นหรือขจัดเศษพิกเซลรอบวัตถุด้วยการดำเนินการสัณฐานวิทยา
    *   การสร้างหน้ากากเส้นขอบ (Binary Edge Mask)
*   **เครื่องมือหลัก**: VS Code, Miniconda
*   **ฟังก์ชัน OpenCV**: `cv2.Canny()`, `cv2.morphologyEx()`, `cv2.getStructuringElement()`
*   **งานปฏิบัติ**: ออกแบบโปรแกรมคัดแยกวัตถุจากขอบภาพ โดยใช้ Morphological operations ช่วยกำจัดเศษฝุ่นผงรอบตัววัตถุเป้าหมาย เพื่อทำให้โครงสร้างภาพขาวดำสะอาดชัดเจน

### สัปดาห์ที่ 5: การตรวจจับโครงร่างและการสกัดภาพแยกชิ้นอัตโนมัติ (Contour Detection)
*   **บรรยาย (2 ชั่วโมง)**:
    *   ทฤษฎีการหาขอบเขตและแยกวัตถุออกจากพื้นหลังด้วย Contour Detection
    *   การสกัดคุณลักษณะของเส้นขอบ (Moments, Area, Perimeter)
    *   การสร้างขอบเขตรอบวัตถุ (Bounding Box & Minimum Area Rectangle) เพื่อส่งต่อให้โมเดล AI ตรวจสอบเฉพาะจุด
*   **ปฏิบัติการ (3 ชั่วโมง) - LAB 5: Smart ROI Dataset Auto-Cropper**:
    *   เขียนโปรแกรมใน VS Code ค้นหาตำแหน่งโครงร่างของวัตถุทั้งหมดในรูปภาพ
    *   คำนวณขนาดและกรองวัตถุขนาดเล็กเกินไปทิ้ง จากนั้นสั่งทำการ Crop วัตถุแยกออกมาทีละภาพโดยอัตโนมัติ
*   **เครื่องมือหลัก**: VS Code, Miniconda
*   **ฟังก์ชัน OpenCV**: `cv2.findContours()`, `cv2.drawContours()`, `cv2.boundingRect()`, `cv2.moments()`
*   **งานปฏิบัติ**: พัฒนาเครื่องมือ Crop วัตถุอัตโนมัติ (Auto-cropper script) โดยโปรแกรมจะแยกแยะวัตถุจากสายพานแล้วสับบันทึกรูปภาพสิ่งของออกเป็นไฟล์เดี่ยว ๆ ลงในโฟลเดอร์ระบุเพื่อนำไปสร้าง Dataset ในสัปดาห์ถัด ๆ ไป

### สัปดาห์ที่ 6: การประมวลผลในโดเมนความถี่และการซ่อมแซมภาพ (Fourier & Inpainting)
*   **บรรยาย (2 ชั่วโมง)**:
    *   การเปลี่ยนภาพจาก Spatial Domain ไปสู่ Frequency Domain ด้วย 2D Discrete Fourier Transform (DFT)
    *   การจำลองสเปกตรัมความถี่และฟิลเตอร์กรองความถี่ต่ำ/สูง (Low-pass/High-pass filter)
    *   ทฤษฎีการซ่อมแซมภาพเสียหายด้วย Image Inpainting เพื่อลบรอยขีดเขียน คราบเปื้อน หรือวัตถุบดบัง
    *   การบีบอัดไฟล์ภาพ (JPEG Compression) และตัวชี้วัดความต่าง (MSE, PSNR)
*   **ปฏิบัติการ (3 ชั่วโมง) - LAB 6: Photo Inpainting and Denoising Tool**:
    *   การเขียนโค้ดแปลงภาพเป็นสเปกตรัมความถี่ วิเคราะห์สัญญาณแทรกกวน และประยุกต์ลบสิ่งรบกวนนั้นออก
    *   การซ่อมแซมรูปภาพที่มีรอยแตกหรือตัวอักษรลายน้ำทับหน้าวัตถุด้วย `cv2.inpaint()`
*   **เครื่องมือหลัก**: VS Code, Miniconda
*   **ฟังก์ชัน OpenCV**: `cv2.inpaint()`, `cv2.dft()`, `cv2.idft()`, `cv2.imencode()`
*   **งานปฏิบัติ**: เขียนโปรแกรมรับภาพที่มีแถบลายน้ำทับ และใช้ฟังก์ชันตรวจจับหน้ากากรอยสีเพื่อซ่อมแซมและลบลายน้ำทิ้งอัตโนมัติแบบเนียนเป็นธรรมชาติ

### สัปดาห์ที่ 7: การดึงจุดเด่นของภาพและการจับคู่ฟีเจอร์แบบดั้งเดิม (Classical Feature Matching)
*   **บรรยาย (2 ชั่วโมง)**:
    *   แนวคิดการทำงานของ Keypoints, Descriptors และ Feature Matching
    *   อัลกอริทึมจุดสำคัญดั้งเดิม: SIFT และ ORB (ความสามารถในการทนทานการเอียงและขนาดเปลี่ยน)
    *   การค้นหาและจับคู่ความคล้ายคลึงระหว่างภาพเปรียบเทียบด้วย Brute-Force Matcher
    *   การหา Perspective Homography เพื่อล้อมกรอบวัตถุที่บิดเบี้ยว
*   **ปฏิบัติการ (3 ชั่วโมง) - LAB 7: Target Object Matching Script**:
    *   เขียนโปรแกรมสกัดจุดเด่นบนรูปสินค้าอ้างอิงด้วย ORB
    *   นำไปจับคู่เปรียบเทียบกับภาพที่ถ่ายในมุมมองกล้อง Webcam แบบสด
    *   วาดโครงร่างเส้นสีแสดงตำแหน่งการสบกันของจุดเด่นเพื่อตีกรอบล้อมรอบวัตถุในกล้องวิดีโอ
*   **เครื่องมือหลัก**: VS Code, Miniconda
*   **ฟังก์ชัน OpenCV**: `cv2.ORB_create()`, `cv2.BFMatcher()`, `cv2.findHomography()`, `cv2.perspectiveTransform()`
*   **งานปฏิบัติ**: สร้างสคริปต์สแกนตรวจสอบโลโก้สินค้าจากกล้องวิดีโอเรียลไทม์ หากตรงกับภาพอ้างอิงของร้านค้า ให้ทำการแสดงป้ายชี้กรอบสีระบุชื่อสินค้า

### สัปดาห์ที่ 9: การก้าวเข้าสู่โมเดลประสาทเทียมและการจำแนกภาพแรกด้วย PyTorch

## 6. โครงสร้างไฟล์ควบคุมสภาพแวดล้อม (environment.yml) สำหรับใช้ร่วมกัน
ให้นักศึกษาสร้างไฟล์ชื่อ `environment.yml` ไว้ที่รากโฟลเดอร์โครงการ (Project Root) ใน VS Code ก่อนจะรันคำสั่งสั่งสร้าง Environment ด้วย Miniconda:

```yaml
name: dip_env
channels:
  - pytorch
  - conda-forge
  - defaults
dependencies:
  - python=3.10.12
  - numpy=1.24.3
  - matplotlib=3.7.1
  - opencv=4.6.0
  - pytorch=2.0.1
  - torchvision=0.15.2
  - cpuonly # ปรับเปลี่ยนเป็น pytorch-cuda=11.7 ได้หากเครื่องมี GPU NVIDIA
  - pip
  - pip:
    - ultralytics==8.0.196
    - mediapipe==0.10.7
```

> **ข้อแนะนำป้องกันบัค**: การกำหนดช่องทางดาวน์โหลด (`channels`) และล็อกเวอร์ชันระดับจุดทศนิยมอย่างชัดเจน จะช่วยให้ผู้เรียนทุกคนดาวน์โหลดไบนารีชนิดเดียวกัน ไม่ว่าจะอยู่บน Windows 10, 11 หรือ macOS ทำให้ฟังก์ชันคำสั่งทั้งหมดเขียนรันผ่าน VS Code ทำงานได้ตรงกันโดยไม่มีอาการเวอร์ชันตีกันหรือติดตั้งแพ็คเกจไม่ผ่าน
