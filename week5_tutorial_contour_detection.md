# เนื้อหาเจาะลึก: การตรวจจับโครงร่างและการสกัดภาพแยกชิ้นอัตโนมัติ (Contour Detection & Auto-Cropper)

ในสัปดาห์ที่ 4 เราได้เรียนรู้วิธีการหาเส้นขอบ (Edge Detection) ด้วยอัลกอริทึม Canny ไปแล้ว แต่เส้นขอบที่ได้มักจะเป็นเส้นขาดๆ ทับซ้อนกัน หรือไม่เป็นรูปร่างที่ปิดสมบูรณ์ ในสัปดาห์นี้ เราจะมารู้จักกับ **Contour (โครงร่าง)** ซึ่งเป็นเทคนิคที่เชื่อมต่อพิกัดทั้งหมดให้กลายเป็นรูปทรงที่ปิดสนิท (Closed shape) ทำให้เราสามารถคำนวณพื้นที่, จุดศูนย์กลาง, และตีกรอบล้อมรอบวัตถุได้

## 1. Contour แตกต่างจาก Edge อย่างไร?
* **Edge (เส้นขอบ):** คือบริเวณที่สีของภาพมีความแตกต่างกันอย่างฉับพลัน (เช่น สีดำตัดกับสีขาว) ผลลัพธ์ที่ได้จะเป็นแค่เส้นพิกเซล ไม่รับรู้ถึงการเป็น "วัตถุชิ้นเดียวกัน"
* **Contour (โครงร่าง):** คือเส้นโค้งที่เชื่อมต่อจุดพิกัดทั้งหมด (ตามขอบเขต) ที่มีค่าความเข้มสีเท่ากัน การหา Contour จะทำให้คอมพิวเตอร์รับรู้ว่าพิกเซลกลุ่มนี้ล้อมรอบกันจนเกิดเป็น **"วัตถุ 1 ชิ้น"**

> **เปรียบเทียบให้เห็นภาพ:** Edge เหมือนเส้นร่างดินสอที่วาดไม่เสร็จ แต่ Contour เหมือนการเอาปากกาเมจิกตัดเส้นรอบรูปวัตถุจนครบวงจร

---

## 2. ขั้นตอนมาตรฐานก่อนการหา Contour (Preprocessing Pipeline)
การจะให้คอมพิวเตอร์หา Contour ได้แม่นยำ รูปภาพต้องมีความชัดเจนระหว่าง "วัตถุ" กับ "พื้นหลัง" ดังนั้นเราจึงต้องผ่าน 3 ขั้นตอนนี้เสมอ:

1. **แปลงเป็นขาวดำ (Grayscale):** ลดมิติข้อมูลสีจาก 3 Channels (BGR) เหลือ 1 Channel เพื่อความรวดเร็วและลดภาระการคำนวณ
2. **เบลอภาพ (Blurring):** ใช้ `cv2.GaussianBlur` เพื่อลบสัญญาณรบกวน (Noise) หรือความขรุขระบนพื้นผิววัตถุ ป้องกันไม่ให้ OpenCV เข้าใจผิดว่าจุดเล็กๆ เหล่านั้นคือขอบเขตวัตถุ
3. **แบ่งแยกสีให้ขาด (Thresholding):** แปลงภาพให้เหลือแค่ 2 สี คือ ดำ (0) กับ ขาว (255) 
   * *ข้อควรจำ:* ฟังก์ชัน FindContours ของ OpenCV จะ **ค้นหาพิกเซลสีขาวที่อยู่บนพื้นหลังสีดำเสมอ**
   * หากวัตถุของคุณมีสีเข้มและพื้นหลังสีสว่าง คุณต้องใช้ `cv2.THRESH_BINARY_INV` เพื่อกลับสีให้วัตถุกลายเป็นสีขาว

## 3. เจาะลึกฟังก์ชันที่สำคัญใน OpenCV

### 3.1 `cv2.findContours()`
ฟังก์ชันหลักสำหรับการสกัดโครงร่าง
```python
contours, hierarchy = cv2.findContours(image, mode, method)
```
**พารามิเตอร์ที่ต้องรู้ (ระดับ ป.ตรี ควรเลือกใช้ให้เป็น):**
* **`mode` (โหมดการดึงความสัมพันธ์):**
  * `cv2.RETR_EXTERNAL`: เอาเฉพาะขอบเขตนอกสุดเท่านั้น (เหมาะสำหรับการนับจำนวนชิ้น หรือการ Crop วัตถุทึบ)
  * `cv2.RETR_TREE`: ดึงโครงร่างทั้งหมดพร้อมจัดลำดับชั้น (Hierarchy) พ่อ-ลูก เช่น การตรวจจับกรอบป้ายทะเบียน และมีตัวอักษรอยู่ข้างใน (ตัวอักษรจะเป็นลูกของป้ายทะเบียน)
* **`method` (วิธีการจัดเก็บพิกัด):**
  * `cv2.CHAIN_APPROX_NONE`: จัดเก็บทุกพิกัดของขอบเขต (หากภาพระดับ 4K วัตถุชิ้นใหญ่อาจมีพิกัดนับหมื่นจุด กินหน่วยความจำมาก)
  * `cv2.CHAIN_APPROX_SIMPLE`: บีบอัดข้อมูลโดยเก็บเฉพาะพิกัดจุดหักมุม เช่น สี่เหลี่ยม 1 รูป จะเก็บพิกัดแค่ 4 จุด (มุมซ้ายขวาบนล่าง) ทำให้ประมวลผลเร็วขึ้นมหาศาล

### 3.2 การหาขนาดพื้นที่ (Area Filtering)
ในโลกความจริง กล้องมักจะจับเศษฝุ่นหรือเงาสะท้อนติดมาด้วย เราใช้ฟังก์ชันนี้เพื่อหาขนาดพื้นที่ของรูปร่าง (มีหน่วยเป็นพิกเซล)
```python
area = cv2.contourArea(contour)
if area > 1000:  # ถ้าพื้นที่เล็กกว่า 1000 พิกเซล ให้ถือว่าเป็นขยะ (Noise) ตัดทิ้งได้เลย
    ...
```

### 3.3 Bounding Box vs Minimum Area Rect (การตีกรอบล้อมวัตถุ)
เมื่อเราได้เส้น Contour ที่หยึกหยักตามรูปทรงวัตถุ การจะสั่งตัดรูป (Crop) ด้วยกรอบหยึกหยักนั้นทำได้ยาก เราจึงต้องตีกรอบครอบมันอีกชั้น ซึ่งมี 2 รูปแบบหลัก:

1. **Straight Bounding Rectangle (`cv2.boundingRect`)**:
   * หากรอบสี่เหลี่ยมแบบตั้งตรง (ขนานกับแกน X, Y ของภาพ) 
   * **ข้อดี:** นำไป Slicing (หั่นภาพ) ลง Array ทันทีได้ง่ายมาก เหมาะสำหรับทำ Auto-Cropper
   * **ผลลัพธ์:** พิกัดมุมซ้ายบน, ความกว้าง, ความสูง `x, y, w, h = cv2.boundingRect(contour)`

2. **Rotated Rectangle (`cv2.minAreaRect`)**:
   * หากรอบสี่เหลี่ยมที่มีขนาดพื้นที่เล็กที่สุด โดยตัวกรอบสามารถ "หมุน" เอียงตามรูปทรงของวัตถุได้
   * **ข้อดี:** วัดขนาด กว้าง x ยาว ของวัตถุที่วางเอียงอยู่ได้แม่นยำกว่า
   * **ผลลัพธ์:** ได้ค่าจุดศูนย์กลาง `(x,y)`, ขนาด `(w,h)`, และองศาการหมุน `angle`

## 4. LAB 5: การพัฒนาโปรแกรมตัดแยกวัตถุอัตโนมัติ (Smart ROI Auto-Cropper)

ในปฏิบัติการนี้ เราจะประยุกต์ใช้ความรู้ด้านบนมาเขียนระบบ Automation รับภาพที่มีวัตถุหลายชิ้น (เช่น ภาพอะไหล่บนโต๊ะ) ให้คอมพิวเตอร์ค้นหาวัตถุ ตีกรอบตั้งตรง และสับภาพย่อย (Crop) ออกมาเซฟเป็นไฟล์ใหม่ทีละชิ้นโดยอัตโนมัติ เพื่อเตรียมทำเป็น Dataset ป้อนให้โมเดล AI ในสัปดาห์ถัดๆ ไป

### โค้ดฉบับสมบูรณ์ พร้อมคำอธิบาย
ให้นักศึกษาสร้างไฟล์ `lab5_auto_cropper.py` ใน VS Code แล้วพิมพ์โค้ดตามด้านล่าง:

```python
import cv2
import numpy as np
import os

# ==========================================
# 1. โหลดภาพและเตรียมโฟลเดอร์สำหรับเซฟ (Setup)
# ==========================================
# โหลดภาพต้นฉบับ (สมมติชื่อไฟล์ objects_on_belt.jpg)
img = cv2.imread('objects_on_belt.jpg')

# เช็คว่าโหลดภาพสำเร็จหรือไม่
if img is None:
    print("ไม่พบไฟล์รูปภาพ กรุณาตรวจสอบชื่อไฟล์และ Path ให้ถูกต้อง")
    exit()

# สร้างโฟลเดอร์สำหรับเก็บภาพที่ถูกตัด (ถ้ายากลับมารันซ้ำ จะได้เซฟลงที่เดิมได้เลย)
output_dir = "cropped_dataset"
if not os.path.exists(output_dir):
    os.makedirs(output_dir)

# ทำสำเนาภาพไว้สำหรับวาดกรอบเพื่อดูผลลัพธ์ตอนจบ (จะไม่วาดกรอบทับภาพต้นฉบับเด็ดขาด)
output_img = img.copy()

# ==========================================
# 2. กระบวนการเตรียมภาพ (Preprocessing)
# ==========================================
# 2.1 แปลงเป็น Grayscale ลดมิติข้อมูลสี
gray = cv2.cvtColor(img, cv2.COLOR_BGR2GRAY)

# 2.2 เบลอภาพลด Noise ด้วย Gaussian (kernel 5x5)
blurred = cv2.GaussianBlur(gray, (5, 5), 0)

# 2.3 ทำ Thresholding: แยกวัตถุออกจากพื้นหลัง
# ใช้ THRESH_BINARY_INV สมมติว่าพื้นหลังสว่างและวัตถุทึบ (กลับสีให้วัตถุเป็นสีขาว)
# หากทดลองรันแล้ววัตถุไม่ถูกจับ ให้ปรับค่า Threshold '150' ขึ้นลงตามสภาพแสง
_, thresh = cv2.threshold(blurred, 150, 255, cv2.THRESH_BINARY_INV)

# ==========================================
# 3. ค้นหา Contour (Find Contours)
# ==========================================
# ใช้ RETR_EXTERNAL เพื่อเอาเฉพาะชิ้นส่วนนอกสุด ไม่สนใจรูพรุนข้างในวัตถุ
contours, hierarchy = cv2.findContours(thresh, cv2.RETR_EXTERNAL, cv2.CHAIN_APPROX_SIMPLE)

print(f"สแกนเจอวัตถุทั้งหมด (รวมชิ้นส่วนขยะ) จำนวน: {len(contours)} ชิ้น")

# ==========================================
# 4. วนลูป กรองขนาด ตีกรอบ และตัดภาพ (Auto-Crop Logic)
# ==========================================
object_count = 0  # ตัวแปรสำหรับตั้งชื่อไฟล์รันนัมเบอร์

for contour in contours:
    # 4.1 คำนวณขนาดพื้นที่
    area = cv2.contourArea(contour)
    
    # 4.2 ตรรกะการกรอง: ตั้งค่า Threshold ของพื้นที่เพื่อตัดฝุ่นขยะทิ้ง
    if area > 800:
        object_count += 1
        
        # 4.3 ตีกรอบ Bounding Box แบบตั้งตรง
        x, y, w, h = cv2.boundingRect(contour)
        
        # 4.4 ตีกรอบสีเขียวลงบนภาพ Output copy เพื่อแสดงให้ User เห็นความสำเร็จ
        # cv2.rectangle(ภาพ, จุดมุมซ้ายบน, จุดมุมขวาล่าง, สี BGR, ความหนาเส้น)
        cv2.rectangle(output_img, (x, y), (x + w, y + h), (0, 255, 0), 2)
        
        # 4.5 กระบวนการตัดภาพ (Crop) 
        # ใช้หลักการ Array Slicing ของ NumPy: image[row_start:row_end, col_start:col_end]
        cropped_object = img[y:y+h, x:x+w]
        
        # 4.6 ประกอบ Path ปลายทางและเซฟภาพ
        save_path = os.path.join(output_dir, f"object_{object_count}.jpg")
        cv2.imwrite(save_path, cropped_object)
        
        # วาดข้อความหมายเลขกำกับไว้บนกรอบ
        cv2.putText(output_img, f"Obj {object_count}", (x, y - 10), 
                    cv2.FONT_HERSHEY_SIMPLEX, 0.5, (0, 255, 0), 2)

print(f"สำเร็จ! ทำการหั่นภาพและบันทึกลงโฟลเดอร์ {output_dir} จำนวน {object_count} ไฟล์")

# ==========================================
# 5. แสดงผลรูปภาพเปรียบเทียบ
# ==========================================
# ลดขนาดหน้าจอแสดงผลหากภาพต้นฉบับมีขนาดใหญ่เกินไป
cv2.imshow("1. Original", cv2.resize(img, (800, 600)))
cv2.imshow("2. Binary Threshold", cv2.resize(thresh, (800, 600)))
cv2.imshow("3. Final Auto-Crop", cv2.resize(output_img, (800, 600)))

cv2.waitKey(0)
cv2.destroyAllWindows()
```

## 5. ข้อควรระวังและ Error ที่พบบ่อย (Troubleshooting & Pitfalls)

1. **ทำไมหา Contour ไม่เจอ หรือเจอกรอบเบ้อเริ่มคลุมทั้งหน้าจอ?**
   * **สาเหตุ:** OpenCV จะหา Contour จาก **สีขาว** ถ้ารูปตอนทำ Threshold ของคุณได้พื้นหลังเป็นสีขาว และวัตถุเป็นสีดำ OpenCV จะมองว่าพื้นหลังทั้งหมดนั่นแหละคือวัตถุ 1 ชิ้นขนาดใหญ่
   * **วิธีแก้:** ต้องใช้โหมดสลับสี `cv2.THRESH_BINARY_INV` เพื่อให้วัตถุกลายเป็นสีขาว และพื้นหลังดำสนิท
2. **ทำไมตัดรูปออกมาแล้วมีขนาดเล็กเกินไป หรือมีวัตถุเกินมาเยอะมาก?**
   * **สาเหตุ:** ไม่ได้ทำ `cv2.GaussianBlur` ก่อน ทำให้เศษฝุ่น แสงสะท้อน ถูกมองว่าเป็นวัตถุ หรือตั้งค่ากรองพื้นที่ (`area > 800`) ต่ำเกินไป
   * **วิธีแก้:** เพิ่มค่าตัวเลขใน `if area > 800:` ให้สูงขึ้นจนกว่าขยะจะหายไป หรือเบลอภาพให้หนักขึ้น `(5, 5) -> (9, 9)`
3. **Array Slicing Error (หน้าจอแสดงผล Crop แหว่ง):**
   * **สาเหตุ:** ลำดับแกน X, Y ใน OpenCV กับ NumPy สวนทางกัน `cv2.boundingRect` ให้ค่า `x, y` แต่เวลาตัดรูปด้วย NumPy ต้องใช้ลำดับ `[row, col]` ซึ่งก็คือ `[y, x]`
   * **วิธีแก้:** จำไว้เสมอว่าสูตรสับรูปคือ `image[y : y+h, x : x+w]` ห้ามสลับเป็น `[x:x+w, y:y+h]` เด็ดขาด
