# เนื้อหาเจาะลึก: การตรวจจับโครงร่างและการสกัดภาพแยกชิ้นอัตโนมัติ (Contour Detection & Auto-Cropper)

ในสัปดาห์ที่ 4 เราได้เรียนรู้วิธีการหาเส้นขอบ (Edge Detection) ด้วยอัลกอริทึม Canny ไปแล้ว แต่เส้นขอบที่ได้มักจะเป็นเส้นขาดๆ ทับซ้อนกัน หรือไม่เป็นรูปร่างที่ปิดสมบูรณ์ ในสัปดาห์นี้ เราจะมารู้จักกับ **Contour (โครงร่าง)** ซึ่งเป็นเทคนิคที่เชื่อมต่อพิกัดทั้งหมดให้กลายเป็นรูปทรงที่ปิดสนิท (Closed shape) ทำให้เราสามารถคำนวณพื้นที่, จุดศูนย์กลาง, และตีกรอบล้อมรอบวัตถุได้

---

## 1. Contour แตกต่างจาก Edge อย่างไร?
* **Edge (เส้นขอบ):** คือบริเวณที่สีของภาพมีความแตกต่างกันอย่างฉับพลัน (เช่น สีดำตัดกับสีขาว) ผลลัพธ์ที่ได้จะเป็นแค่เส้นพิกเซล ไม่รับรู้ถึงการเป็น "วัตถุชิ้นเดียวกัน"
* **Contour (โครงร่าง):** คือเส้นโค้งที่เชื่อมต่อจุดพิกัดทั้งหมด (ตามขอบเขต) ที่มีค่าความเข้มสีเท่ากัน การหา Contour จะทำให้คอมพิวเตอร์รับรู้ว่าพิกเซลกลุ่มนี้ล้อมรอบกันจนเกิดเป็น **"วัตถุ 1 ชิ้น"**

> **เปรียบเทียบให้เห็นภาพ:** Edge เหมือนเส้นร่างดินสอที่วาดไม่เสร็จ แต่ Contour เหมือนการเอาปากกาเมจิกตัดเส้นรอบรูปวัตถุจนครบวงจร

---

## 2. ขั้นตอนมาตรฐานก่อนการหา Contour (Preprocessing Pipeline)
การจะให้คอมพิวเตอร์หา Contour ได้แม่นยำ รูปภาพต้องมีความชัดเจนระหว่าง "วัตถุ" กับ "พื้นหลัง" ดังนั้นเราจึงต้องผ่าน 3 ขั้นตอนนี้เสมอ:

1. **แปลงเป็นขาวดำ (Grayscale):** ลดข้อมูลสีเพื่อความรวดเร็วและง่ายต่อการประมวลผล
2. **เบลอภาพ (Blurring):** ใช้ `cv2.GaussianBlur` เพื่อลบสัญญาณรบกวน (Noise) หรือเศษฝุ่นเล็กๆ ออกไป ป้องกันการเกิด Contour ขยะ
3. **แบ่งแยกชัดเจน (Thresholding หรือ Canny Edge):** แปลงภาพให้เหลือแค่ 2 สี คือ ดำ (พื้นหลัง) กับ ขาว (วัตถุเป้าหมาย) 

---

## 3. ฟังก์ชันที่สำคัญใน OpenCV

### 3.1 `cv2.findContours()`
ฟังก์ชันสำหรับค้นหาพิกัดทั้งหมดที่เป็น Contour
```python
contours, hierarchy = cv2.findContours(image, mode, method)
```
* **image:** ภาพขาวดำ (Binary Image) ที่ผ่านการทำ Threshold มาแล้ว
* **mode:** โหมดการดึง Contour ที่นิยมใช้คือ `cv2.RETR_EXTERNAL` (ดึงเฉพาะเส้นขอบนอกสุด ไม่เอาขอบรูพรุนข้างในวัตถุ) หรือ `cv2.RETR_TREE` (ดึงทั้งหมดแบบมีลำดับชั้น พ่อ-ลูก)
* **method:** วิธีการเก็บพิกัด นิยมใช้ `cv2.CHAIN_APPROX_SIMPLE` เพราะจะเก็บเฉพาะจุดมุมที่จำเป็น (เช่น สี่เหลี่ยมเก็บแค่ 4 มุม) แทนที่จะเก็บทุกพิกัดเรียงกัน ช่วยประหยัดหน่วยความจำ
* **ผลลัพธ์ (contours):** จะได้ข้อมูลเป็น List ของ Array ที่เก็บพิกัดขอบเขตของแต่ละวัตถุไว้

### 3.2 `cv2.contourArea(contour)`
หาพื้นที่ (ขนาด) ของ Contour นั้นๆ มีประโยชน์มากในการกรอง (Filter) เอาเศษฝุ่นหรือวัตถุที่ไม่ต้องการทิ้งไป โดยดูจากขนาดที่เล็กเกินไป

### 3.3 `cv2.boundingRect(contour)`
ฟังก์ชันพระเอกของการทำ Auto-Cropper มันจะคำนวณและคืนค่า **กรอบสี่เหลี่ยมตั้งตรงที่เล็กที่สุด** ที่สามารถครอบ Contour ตัวนี้ได้
* จะคืนค่ามา 4 ตัว: `x, y, w, h` (พิกัดมุมซ้ายบนแกน X, Y และ ความกว้าง Width, ความสูง Height)

## 4. LAB 5: การพัฒนาโปรแกรมตัดแยกวัตถุอัตโนมัติ (Smart ROI Auto-Cropper)
ในปฏิบัติการนี้ เราจะเขียนโปรแกรมรับภาพที่มีวัตถุหลายชิ้น (เช่น ภาพอะไหล่บนโต๊ะ หรือภาพสินค้าบนสายพาน) ให้คอมพิวเตอร์ค้นหาวัตถุ ตีกรอบ และตัดภาพย่อย (Crop) ออกมาเซฟเป็นไฟล์ใหม่ทีละชิ้นโดยอัตโนมัติ เพื่อนำไปทำ AI Dataset ต่อไป

### โค้ดฉบับสมบูรณ์ พร้อมคำอธิบาย
ให้นักศึกษาสร้างไฟล์ `lab5_auto_cropper.py` ใน VS Code แล้วพิมพ์โค้ดตามด้านล่าง:

```python
import cv2
import numpy as np
import os

# ==========================================
# 1. โหลดภาพและเตรียมโฟลเดอร์สำหรับเซฟ (Setup)
# ==========================================
# โหลดภาพต้นฉบับ (สมมติชื่อไฟล์ objects_on_belt.jpg)
img = cv2.imread('objects_on_belt.jpg')

# เช็คว่าโหลดภาพสำเร็จหรือไม่
if img is None:
    print("ไม่พบไฟล์รูปภาพ กรุณาตรวจสอบชื่อไฟล์และ Path")
    exit()

# สร้างโฟลเดอร์สำหรับเก็บภาพที่ถูกตัด หากยังไม่มีโฟลเดอร์นี้
output_dir = "cropped_dataset"
if not os.path.exists(output_dir):
    os.makedirs(output_dir)

# ทำสำเนาภาพไว้สำหรับวาดกรอบเพื่อดูผลลัพธ์ (จะได้ไม่ทับภาพต้นฉบับ)
output_img = img.copy()

# ==========================================
# 2. กระบวนการเตรียมภาพ (Preprocessing)
# ==========================================
# 2.1 แปลงเป็น Grayscale
gray = cv2.cvtColor(img, cv2.COLOR_BGR2GRAY)

# 2.2 เบลอภาพลด Noise
blurred = cv2.GaussianBlur(gray, (5, 5), 0)

# 2.3 ทำ Thresholding: แยกวัตถุออกจากพื้นหลัง
# (สมมติว่าพื้นหลังสว่าง วัตถุทึบ จึงใช้ THRESH_BINARY_INV เพื่อให้วัตถุเป็นสีขาว พื้นเป็นสีดำ)
_, thresh = cv2.threshold(blurred, 150, 255, cv2.THRESH_BINARY_INV)

# สามารถใช้ Canny Edge แทน Thresholding ก็ได้เช่นกัน (ขึ้นอยู่กับภาพ)
# edged = cv2.Canny(blurred, 50, 150)

# ==========================================
# 3. ค้นหา Contour (Find Contours)
# ==========================================
# RETR_EXTERNAL = เอาเฉพาะขอบนอกสุด, CHAIN_APPROX_SIMPLE = เก็บจุดมุมประหยัดเมมโมรี่
contours, hierarchy = cv2.findContours(thresh, cv2.RETR_EXTERNAL, cv2.CHAIN_APPROX_SIMPLE)

print(f"ตรวจพบวัตถุทั้งหมด (รวม Noise) จำนวน: {len(contours)} ชิ้น")

# ==========================================
# 4. วนลูป กรองขนาด ตีกรอบ และตัดภาพ (Auto-Crop Logic)
# ==========================================
object_count = 0  # ตัวแปรนับจำนวนวัตถุที่ใช้งานได้จริง

for contour in contours:
    # 4.1 คำนวณขนาดพื้นที่
    area = cv2.contourArea(contour)
    
    # 4.2 ตั้งค่า Threshold ของพื้นที่เพื่อกรองฝุ่นขยะ (สมมติเอาพื้นที่ที่ใหญ่กว่า 500 พิกเซล)
    if area > 500:
        object_count += 1
        
        # 4.3 หาพิกัด Bounding Box
        x, y, w, h = cv2.boundingRect(contour)
        
        # 4.4 ตีกรอบสีเขียวลงบนภาพ Output เพื่อแสดงให้ User เห็น
        # cv2.rectangle(ภาพ, จุดซ้ายบน, จุดขวาล่าง, สี BGR, ความหนาเส้น)
        cv2.rectangle(output_img, (x, y), (x + w, y + h), (0, 255, 0), 2)
        
        # 4.5 ทำการตัดภาพ (Crop) จากภาพต้นฉบับ
        # ใช้หลักการ Array Slicing: image[y_start:y_end, x_start:x_end]
        cropped_object = img[y:y+h, x:x+w]
        
        # 4.6 เซฟภาพย่อยลงโฟลเดอร์ที่เตรียมไว้
        save_path = os.path.join(output_dir, f"object_{object_count}.jpg")
        cv2.imwrite(save_path, cropped_object)
        
        # ใส่ตัวหนังสือระบุหมายเลขกำกับบนภาพที่ตีกรอบ
        cv2.putText(output_img, f"Obj {object_count}", (x, y - 10), 
                    cv2.FONT_HERSHEY_SIMPLEX, 0.5, (0, 255, 0), 2)

print(f"สำเร็จ! ทำการหั่นภาพและบันทึกลงโฟลเดอร์ {output_dir} จำนวน {object_count} ไฟล์")

# ==========================================
# 5. แสดงผลรูปภาพเปรียบเทียบ
# ==========================================
cv2.imshow("Original Image", img)
cv2.imshow("Threshold (Binary)", thresh)
cv2.imshow("Detected Objects", output_img)

cv2.waitKey(0)
cv2.destroyAllWindows()
```

### สรุปหลักการทำงานของโค้ด:
1. **เตรียมภาพ (Preprocessing):** เปลี่ยนเป็นภาพที่มีแค่ขาวกับดำ เพื่อให้คอมพิวเตอร์แยกออกว่าอะไรคือพื้นหลัง (สีดำ) และอะไรคือวัตถุ (สีขาว)
2. **หาเส้นขอบ (Find Contours):** โปรแกรมจะสแกนหาขอบเขตสีขาวทั้งหมด
3. **วิเคราะห์และกรอง (Filter):** ในบางครั้งอาจมีฝุ่นหรือแสงสะท้อนที่กลายเป็นจุดสีขาวเล็กๆ เราใช้ `cv2.contourArea(contour) > 500` เพื่อบอกว่า "ถ้าพื้นที่เล็กเกินไป ให้ถือว่าเป็นขยะ ไม่ต้องตัดรูป"
4. **ตีกรอบและหั่นรูป (Bounding Box & Slicing):** ใช้ `cv2.boundingRect()` เพื่อดึงกรอบสี่เหลี่ยม จากนั้นใช้เทคนิค `[y:y+h, x:x+w]` เพื่อหยิบเอาเฉพาะพิกเซลช่วงนั้นใน Array รูปภาพต้นฉบับออกมา และทำการเซฟเป็นไฟล์ใหม่

> **ข้อแนะนำ:** การหา Threshold (`150` ในฟังก์ชัน `cv2.threshold`) อาจต้องมีการปรับแต่งตามสภาพแสงและสีของวัตถุจริงของนักศึกษาแต่ละคน การ Debug ใน VS Code เพื่อดูภาพตัวแปร `thresh` จะช่วยให้หาระดับที่เหมาะสมได้เร็วที่สุด
