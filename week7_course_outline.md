**เวลาเรียน:** 5 ชั่วโมง (บรรยาย 2 ชั่วโมง / ปฏิบัติการ 3 ชั่วโมง)

## 1. วัตถุประสงค์การเรียนรู้ (Learning Objectives)
เมื่อสิ้นสุดการเรียนในสัปดาห์นี้ นักศึกษาจะสามารถ:
1. **อธิบายแนวคิด** ของจุดสำคัญ (Keypoints) และตัวบรรยายลักษณะจุดเด่น (Descriptors) ในรูปภาพได้
2. **เข้าใจความแตกต่าง** ระหว่างอัลกอริทึมจุดสำคัญแบบดั้งเดิมอย่าง SIFT และ ORB ทั้งในแง่ลิขสิทธิ์ ความเร็ว และความแม่นยำ
3. **ใช้งานฟังก์ชัน OpenCV** สำหรับการตรวจหาและจับคู่ฟีเจอร์เด่น (เช่น `cv2.ORB_create()`, `cv2.BFMatcher()`)
4. **คำนวณหา Perspective Homography** เพื่อล้อมกรอบวัตถุเป้าหมายที่เกิดการบิดเบี้ยว หมุน หรือย่อขยายขนาดในภาพจริงได้
5. **พัฒนาโปรแกรมจำแนกโลโก้หรือค้นหาวัตถุ (Target Object Matching Script)** ผ่านการดีบักและการเขียนสคริปต์ใน VS Code

## 2. แผนการสอนช่วงบรรยาย (Lecture Plan) - 2 ชั่วโมง

### 2.1 บทนำสู่ Feature Detection & Description (30 นาที)
* **ทำไมต้องดึงจุดเด่น?**: ข้อจำกัดของการจับคู่ภาพแบบ Pixel-to-Pixel หรือการใช้ Template Matching เมื่อภาพมีการหมุน (Rotation), เปลี่ยนขนาด (Scaling), หรือเปลี่ยนมุมกล้อง (Perspective)
* **นิยามฟีเจอร์เด่น**:
  * **Keypoint (จุดสำคัญ)**: ตำแหน่งพิกเซลที่มีความโดดเด่นสะดุดตา เช่น มุม (Corners) หรือจุดเปลี่ยนสีแบบฉับพลัน
  * **Descriptor (ตัวบรรยายลักษณะ)**: เวกเตอร์คณิตศาสตร์ที่เก็บลักษณะสภาพแวดล้อมรอบๆ จุดสำคัญ เพื่อนำไปใช้เปรียบเทียบกับภาพอื่น

### 2.2 อัลกอริทึม SIFT และ ORB (30 นาที)
* **SIFT (Scale-Invariant Feature Transform)**: หลักการทำงานดั้งเดิมที่ทนทานสูงต่อขนาดและแสง แต่ประมวลผลช้าและเคยติดลิขสิทธิ์
* **ORB (Oriented FAST and Rotated BRIEF)**: ทางเลือกแบบ Open-source ที่พัฒนาโดย OpenCV เน้นประมวลผลเร็ว รองรับ Real-time และทนทานการหมุน

### 2.3 การจับคู่ฟีเจอร์และคณิตศาสตร์ Homography (45 นาที)
* **Feature Matching**: Brute-Force Matcher (การค้นหาเพื่อนบ้านใกล้สุด) และการคัดกรองคู่ที่ถูกต้องด้วย Ratio Test ของ Lowe
* **Homography ($H$)**: แมทริกซ์การแปลงพิกัดมิติ $3 \times 3$ ที่เปลี่ยนระนาบ 2 มิติหนึ่งไปยังอีกระนาบหนึ่ง ($x' = Hx$)
* **RANSAC (Random Sample Consensus)**: อัลกอริทึมขจัดจุดจับคู่ที่ผิดพลาด (Outliers) ออกจากการคำนวณ Homography

### 2.4 ถาม-ตอบและสรุปบทเรียน (15 นาที)
* ซักถามความเข้าใจเกี่ยวกับกลไกการจับคู่ และการประยุกต์ใช้ในการระบุพิกัดวัตถุก่อนเข้าภาคปฏิบัติ

## 3. แผนการสอนช่วงปฏิบัติการ (Lab Plan) - 3 ชั่วโมง
**Lab 7: Target Object Matching Script**

### 3.1 การเตรียมสภาพแวดล้อมและจัดเตรียมรูปภาพ (30 นาที)
* เปิด VS Code และเตรียมความพร้อมสภาพแวดล้อม `dip_env` ที่สร้างไว้
* เตรียมรูปภาพอ้างอิงของวัตถุเดี่ยว (เช่น รูปภาพโลโก้แบรนด์ หรือภาพปกหนังสือตั้งตรงเรียบๆ) และภาพสภาพแวดล้อมจริงที่มีวัตถุนั้นปะปนอยู่ (เช่น ภาพชั้นวางสินค้าที่มีมุมเอียงหรือแสงเปลี่ยนไป)

### 3.2 ตรวจจับและสกัด Descriptor ด้วย ORB (45 นาที)
* เขียนสคริปต์สกัดจุดสำคัญ (Keypoints) และตัวบรรยาย (Descriptors) ของภาพทั้งสองภาพโดยใช้ `cv2.ORB_create()`
* วาดจุดเด่นที่ตรวจพบลงบนภาพด้วย `cv2.drawKeypoints()` เพื่อวิเคราะห์การกระจายตัวของฟีเจอร์เด่น

### 3.3 จับคู่จุดเด่นด้วย BFMatcher (45 นาที)
* สร้าง Object `cv2.BFMatcher` เพื่อหาจุดคู่สบของ Descriptors ระหว่างสองภาพ
* ประยุกต์ใช้เทคนิค K-Nearest Neighbor (KNN) Match และคำนวณ Distance Ratio Test เพื่อคัดสรรเฉพาะคู่จับคู่คุณภาพสูง (Good Matches)
* แสดงผลลัพธ์การจับคู่ด้วย `cv2.drawMatches()`

### 3.4 คำนวณหาพิกัดพื้นที่และวาดกรอบสีเขียวล้อมรอบเป้าหมาย (60 นาที)
* แปลงตำแหน่งพิกัดของ Good Matches เป็น NumPy Array
* ใช้ฟังก์ชัน `cv2.findHomography()` ร่วมกับ `cv2.RANSAC` เพื่อคำนวณแมทริกซ์การทรานส์ฟอร์ม
* คำนวณขอบเขตมุมทั้ง 4 ของภาพอ้างอิง แล้วเปลี่ยนตำแหน่งไปยังภาพปลายทางด้วย `cv2.perspectiveTransform()`
* ใช้ `cv2.polylines()` วาดกรอบสี่เหลี่ยมสีเขียวล้อมรอบวัตถุในภาพสภาพแวดล้อมจริงเพื่อยืนยันพิกัดที่ถูกต้อง
